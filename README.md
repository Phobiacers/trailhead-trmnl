# Trailhead for TRMNL

Trailhead is a private TRMNL plugin that rotates a curated off-road trail discovery card once per refresh. It is built for the polling strategy: TRMNL fetches static JSON from GitHub Pages, then the Liquid layouts render a monochrome route trace, elevation profile, stats, and a marquee obstacle callout.

The starter catalog includes Arizona, California, Colorado, and Utah with five trails each. The data model is ready to expand toward the 15-20 trails-per-state target.

## What is included

- `trmnl-import/` contains the flat private plugin files: `settings.yml`, `full.liquid`, `half_horizontal.liquid`, `half_vertical.liquid`, and `quadrant.liquid`.
- `catalog/trails.json` is the editable trail catalog.
- `public/api/` is the generated static JSON API for GitHub Pages.
- `.github/workflows/pages.yml` deploys the static API to GitHub Pages on push and refreshes it daily.
- `worker/` is an optional Cloudflare Worker version if you ever want live request-time filtering.
- `dist/trailhead-private-plugin.zip` is generated by `npm run build` and can be imported into TRMNL.

## Build

```sh
npm run build
```

This generates `public/api/`, validates the catalog/templates, and creates:

```text
dist/trailhead-private-plugin.zip
```

## Push to GitHub Pages

1. Create an empty GitHub repo. A good name is `trailhead-trmnl`.
2. Push this folder to GitHub:

```sh
git init
git add .
git commit -m "Initial Trailhead TRMNL plugin"
git branch -M main
git remote add origin https://github.com/Phobiacers/trailhead-trmnl.git
git push -u origin main
```

3. In the GitHub repo, open Settings > Pages.
4. Set Source to GitHub Actions.
5. Run the "Deploy GitHub Pages API" workflow, or push to `main` and let it run.
6. Your polling JSON will be available at:

```text
https://Phobiacers.github.io/trailhead-trmnl/api/ut/difficulty-8/seasonal-true.json
```

## Configure the TRMNL import ZIP

Update the placeholder in `trmnl-import/settings.yml`:

```text
https://Phobiacers.github.io/trailhead-trmnl
```

Then rebuild the ZIP:

```sh
npm run build
```

The TRMNL polling URL uses form fields to pick the static JSON file:

- `state`: `az`, `ca`, `co`, or `ut`
- `max_difficulty`: `1` through `10`
- `seasonal_only`: `true` or `false`

Inside TRMNL, use the saved custom field values explicitly:

```text
https://Phobiacers.github.io/trailhead-trmnl/api/{{ trmnl.plugin_settings.custom_fields_values.state | default: 'ca' }}/difficulty-{{ trmnl.plugin_settings.custom_fields_values.max_difficulty | default: '10' }}/seasonal-{{ trmnl.plugin_settings.custom_fields_values.seasonal_only | default: 'true' }}.json
```

Example:

```text
https://Phobiacers.github.io/trailhead-trmnl/api/ut/difficulty-8/seasonal-true.json
```

The GitHub Action rebuilds those JSON files every day at 10:10 UTC so the daily trail rotation keeps moving without a server.

## Import into TRMNL

1. In TRMNL, create or import a Private Plugin.
2. Import `dist/trailhead-private-plugin.zip`.
3. Pick a state, max difficulty, and seasonal filter in the plugin settings.
4. Force refresh once to confirm the polling payload appears in the markup editor.

## Optional Worker endpoint

The Cloudflare Worker version is still available for request-time filtering:

```sh
npm run deploy:worker
```

GitHub Pages is the default path for this repo.

## Data philosophy

The route trace and elevation profile are display glyphs, not maps. They are meant to survive TRMNL's small monochrome display and make the selected obstacle legible at a glance.

Do not scrape Trails Offroad, onX, or other proprietary trail guides. Expand the catalog from public land-manager maps, open geodata, permissively licensed tracks, local club knowledge you can publish, and your own route research. See `docs/data-notes.md` for the schema and curation checklist.
